# 打印机调研记录

## 已知信息

- 已提供 Zebra Link-OS SDK v4.0.3435。
- 截图显示使用 `demos-desktop/Source/DeveloperDemo_Windows.sln`。
- 项目为 C# WPF demo，目标框架为 `net9.0-windows10.0.19041`。
- SDK 文档说明测试系统为 64 位 Windows 10/11，开发环境为 .NET 9、Visual Studio 2022 17.14+。
- 打印机不支持网口，连接方式为 USB。
- 现场反馈未主动安装驱动，认为是免驱。
- 现有 demo 已修改 `SendFileView.xaml.cs`，直接写 ZPL 字符串并发送字节流，成功打印出一维码。

## 当前代码逻辑

核心流程：

```csharp
printerConnection = connectionSelector.GetConnection();
printerConnection.Open();

string zplCode = @"
^XA
^LH100,800
^BY6
^BCN,160,Y,N,N
^FD1234567890^FS
^XZ";

byte[] zplBytes = System.Text.Encoding.UTF8.GetBytes(zplCode);
printerConnection.Write(zplBytes);
printerConnection.Close();
```

## 对当前结果的判断

当前验证重点不是 Zebra SDK 高级模板功能，而是：

- 能建立 USB 打印连接。
- 能发送原始 ZPL 字节流。
- 打印机能够识别 ZPL 并输出一维码。

这说明打印模块可以简化为“动态生成 ZPL + 发送字节流”。

## 仍需确认的关键点

`printerConnection.Write(...)` 只是 Zebra SDK 的统一写入接口，不能单凭它判断底层通道。

需要确认 `connectionSelector.GetConnection()` 返回的实际类型：

- `DriverPrinterConnection`：通过 Windows 打印机/驱动队列发送 RAW ZPL。
- `UsbConnection`：通过 USB Direct 直连设备发送。

建议临时加：

```csharp
MessageBox.Show(printerConnection.GetType().FullName);
```

加在：

```csharp
printerConnection = connectionSelector.GetConnection();
MessageBox.Show(printerConnection.GetType().FullName);
printerConnection.Open();
```

## Win7 风险

Zebra SDK v4.0.3435 不适合作为 Win7 最终方案直接定型，原因：

- SDK 项目目标为 Windows 10.0.19041。
- 官方文档仅列出 Windows 10/11。
- 运行依赖 .NET 9，不支持 Win7。
- USB Direct 相关实现可能依赖较新的 Windows API。

如果现场能升级 Win10，则优先沿用当前 SDK 方案。

如果必须 Win7，则需要做一轮兼容验证：

- Win7 是否能识别打印机为系统打印机。
- 是否能通过 RAW 打印队列发送 ZPL。
- 是否需要安装 Zebra/ZDesigner 驱动。
- 是否需要使用旧版 Zebra SDK 或完全绕开 SDK。

## 建议技术路线

### Win10 路线

- 使用 Zebra SDK 建立 USB 连接。
- 根据业务字段生成 ZPL。
- 使用 `Write(byte[])` 下发打印。
- 使用 SDK 状态接口读取离线、缺纸等状态。

### Win7 路线

- 优先验证 Windows 打印机队列 RAW ZPL。
- 如果打印机未出现在“设备和打印机”，再评估驱动安装或旧版 SDK。
- 尽量避免依赖 v4.0.3435。
