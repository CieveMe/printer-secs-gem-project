# 对接问题清单

## 1. 运行环境

- 已确认：最终运行电脑只考虑 Win10 以上，现场系统由原系统升级到 Win10。
- 已确认：现场会根据本项目要求安装对应 Win10 环境和运行库。
- Win10 是 32 位还是 64 位？如现场可按要求安装，可按最终发布包要求准备。
- 最终程序形态是桌面程序、后台服务，还是命令行工具？

## 2. 打印机

- 打印机准确型号是否为 ZD888TA？
- 打印机只支持 USB，是否确认无网口。
- 当前 demo 是在 Win7 还是 Win10/Win11 上跑通？
- 当前 demo 底层连接对象是什么：
  - `Zebra.Sdk.Comm.DriverPrinterConnection`
  - `Zebra.Sdk.Comm.UsbConnection`
- 建议临时加入以下代码确认：

```csharp
printerConnection = connectionSelector.GetConnection();
MessageBox.Show(printerConnection.GetType().FullName);
printerConnection.Open();
```

- 已确认：标签纸尺寸为 2.5 x 1.575 英寸，约 63.5mm x 40mm。
- 已确认：打印机 DPI 为 203dpi，对应标签画布约 508 x 320 dots。
- 当前样张可见打印字段为 `EFS08IZS` 这类单个字符串，同时作为上方大字、一维码内容和下方可读文本。
- 后续打印内容是否一直只有一个字符串，还是会增加货架号、格口号、RFID、批次、日期、条码/二维码？
- 是否需要中文、Logo、二维码或多种模板？
- 如何判断缺纸、离线、盖子打开、打印完成、打印失败？

## 3. SECS/GEM-HSMS

- 已确认：本程序作为 EQ。
- 已确认：EQ 主动连接 Host。
- Host IP、端口、Device ID 做成 `appsettings.json` 配置项，开发阶段可用默认值/模拟值，真实联调或部署时再填现场值。
- T3/T5/T6/T7/T8 超时参数是多少？
- 是否有对接 Host 模拟器或真实抓包？
- 文档中的 `S5F1` 是否为项目自定义消息？
- 打印指令 `S8F3` 的打印内容等拿到真实报文或样例后先分析；当前按一个字符串生成上方文字、Code128 和下方文字。

## 4. 串口电子货架设备

- 已确认：写 tag 和回传 tag 都是串口操作。
- 已确认：写 tag 对应 Host 下发 RFID/tag，EQ 写入硬件后返回结果。
- 已确认：回传 tag 对应 EQ 读取到 RFID/tag 后主动上报，当前按标准 `S6F11`。
- 已确认：硬件接口集成方式为对方提供 C# 串口硬件代码或 DLL，本项目负责集成调用。
- 等对方提供硬件代码后先分析源码、DLL、示例工程或现有项目片段；能从代码看出来的调用方法、入参、返回值、错误码不再提前追问，分析不出来再补问。
- 串口号如何配置，是否固定 COM 口？
- 波特率、数据位、停止位、校验位分别是多少？
- 帧头、长度、命令字、数据区、校验、帧尾格式是什么？
- 校验算法是 XOR、SUM、CRC16 还是其他？
- 支持哪些命令：查询格口、读 RFID、写 RFID、清空 RFID、状态订阅？
- 是否有主动上报，还是必须轮询？
- 超时、重试、断线重连规则是什么？
- 是否有串口协议文档和真实收发日志？

## 5. 业务流程

- 是 Host 下发 `S8F3` 后立即打印，还是格口/RFID 状态变化触发打印？
- 打印成功后是否需要主动上报 `CE2003`？
- 打印失败后是否需要同时回 `S8F4` 和报警 `AL2002/CE2004`？
- RFID 设置成功后是否需要主动上报 `CE2002`？
- 多货架、多格口、多打印机如何编号和映射？

## 6. 验收

- 交付物是源码、安装包，还是可运行程序？
- 是否需要本地日志、配置界面、操作记录导出？
- 是否需要远程联调或现场联调？
- 验收用例由谁提供，是否包含异常场景？
