# 对接问题清单

## 1. 运行环境

- 最终运行电脑能否升级到 Win10？
- 如果必须 Win7，是 32 位还是 64 位？
- 现场是否允许安装 .NET Framework、.NET Runtime、VC++ Runtime、打印机驱动或其他 USB 驱动？
- 最终程序形态是桌面程序、后台服务，还是命令行工具？

## 2. 打印机

- 打印机准确型号是否为 ZD888TA？
- 打印机只支持 USB，是否确认无网口。
- 当前 demo 是在 Win7 还是 Win10/Win11 上跑通？
- 当前 demo 底层连接对象是什么：
  - `Zebra.Sdk.Comm.DriverPrinterConnection`
  - `Zebra.Sdk.Comm.UsbConnection`
- 建议临时加入以下代码确认：

```csharp
printerConnection = connectionSelector.GetConnection();
MessageBox.Show(printerConnection.GetType().FullName);
printerConnection.Open();
```

- 标签纸尺寸、DPI、横竖方向、打印坐标是否已确定？
- 打印内容有哪些字段：货架号、格口号、RFID、批次、日期、条码/二维码？
- 是否需要中文、Logo、二维码或多种模板？
- 如何判断缺纸、离线、盖子打开、打印完成、打印失败？

## 3. SECS/GEM-HSMS

- 我们的软件最终作为 Host 还是 EQ？
- HSMS 连接由谁主动发起？
- Host/EQ 的 IP、端口、Device ID 是多少？
- T3/T5/T6/T7/T8 超时参数是多少？
- 是否有对接 Host 模拟器或真实抓包？
- 文档中的 `S5F1`、`S6F21` 是否为项目自定义消息？
- 打印指令 `S8F3` 的打印内容是否直接作为条码内容，还是需要解析后重新拼接模板？

## 4. 串口电子货架设备

- 已确认：写 tag 和回传 tag 都是串口操作。
- 已确认：写 tag 对应 Host 下发 RFID/tag，EQ 写入硬件后返回结果。
- 已确认：回传 tag 对应 EQ 读取到 RFID/tag 后主动上报，可用 `S6F11` 或 `S6F21`。
- 串口号如何配置，是否固定 COM 口？
- 波特率、数据位、停止位、校验位分别是多少？
- 帧头、长度、命令字、数据区、校验、帧尾格式是什么？
- 校验算法是 XOR、SUM、CRC16 还是其他？
- 支持哪些命令：查询格口、读 RFID、写 RFID、清空 RFID、状态订阅？
- 是否有主动上报，还是必须轮询？
- 超时、重试、断线重连规则是什么？
- 是否有串口协议文档和真实收发日志？

## 5. 业务流程

- 是 Host 下发 `S8F3` 后立即打印，还是格口/RFID 状态变化触发打印？
- 打印成功后是否需要主动上报 `CE2003`？
- 打印失败后是否需要同时回 `S8F4` 和报警 `AL2002/CE2004`？
- RFID 设置成功后是否需要主动上报 `CE2002`？
- 多货架、多格口、多打印机如何编号和映射？

## 6. 验收

- 交付物是源码、安装包，还是可运行程序？
- 是否需要本地日志、配置界面、操作记录导出？
- 是否需要远程联调或现场联调？
- 验收用例由谁提供，是否包含异常场景？
