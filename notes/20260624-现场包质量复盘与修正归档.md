## 2. 本轮暴露的问题和原因

### 2.1 SDK 误判

错误现象：
- 先用系统 `dotnet` 编译，系统路径是 `C:\Program Files\dotnet\dotnet.exe`。
- 系统 `dotnet` 只有 runtime，没有 SDK，导致误以为需要下载 SDK。

实际事实：
- 本地 SDK 已存在：
  - `D:\Desktop\打印机项目\dotnet-sdk-8.0.421-win-x64\dotnet.exe`
  - `D:\Desktop\打印机项目\.dotnet-sdk\dotnet.exe`
- 两者都能列出 SDK `8.0.421`。

修正：
- `scripts\publish-win-x64.ps1` 已改为优先使用 `dotnet-sdk-8.0.421-win-x64\dotnet.exe`，再 fallback 到 `.dotnet-sdk\dotnet.exe`，最后才用系统 `dotnet`。
- 构建和发布时固定设置工作区内环境变量：
  - `DOTNET_CLI_HOME=D:\Desktop\打印机项目\.dotnet-home`
  - `NUGET_PACKAGES=D:\Desktop\打印机项目\.nuget-packages`
  - `APPDATA=D:\Desktop\打印机项目\.appdata\Roaming`
  - `LOCALAPPDATA=D:\Desktop\打印机项目\.appdata\Local`

### 2.2 发布流程不稳

错误现象：
- 初次整理完整包时，完整包目录曾因为 `Copy-Item -LiteralPath *` 没把文件复制进去。
- 后续人工检查发现并重打包，但说明发布脚本/打包流程缺少硬校验。

修正：
- 发布脚本现在检查完整发布目录必须包含：
  - `PrinterSecsGem.Eq.exe`
  - `App.config`
  - `log4net.config`
  - `stop-printer-secs-gem.cmd`
  - `zebra-command-line/`
  - `secs-simulator/`
- 发布脚本会移除并禁止保留：
  - `PrinterSecsGem.Eq.dll.config`
  - `PrinterSecsGem.Eq.exe.config`
- 原因：客户只应该维护当前目录 `App.config`，不能让客户看到多个配置文件后改错。

### 2.3 App.config 编码问题

用户反馈：
- 客户那边打开配置文件中文乱码。
- 编码显示类似 `utf-8-bom`。

本地检测事实：
- 修正前本地源码、发布目录、zip 内 `App.config` 的首字节是 `3C 3F 78 6D 6C 20 76 65`，即从 `<?xml ve` 开始，不是 `EF BB BF`。
- 但这不代表脚本运行后安全。

真正根因：
- Windows PowerShell 5 对无 BOM UTF-8 文件执行 `Get-Content` 时，默认可能按 ANSI 读取。
- 之前脚本是：
  - `[xml]$xml=Get-Content -LiteralPath $path -Raw`
  - `$xml.Save($path)`
- 这会把中文注释读成乱码后再保存，即使最终文件仍然没有 BOM，也已经内容损坏。

修正：
- 所有写 `App.config` 的脚本读取时改为：
  - `Get-Content -LiteralPath $path -Raw -Encoding UTF8`
- 所有写 `App.config` 的脚本保存时改为显式 UTF-8 无 BOM：
  - `XmlWriterSettings.Encoding = New-Object System.Text.UTF8Encoding -ArgumentList $false`
- C# 新增 `XmlConfigFile.Save(...)`，统一使用 `new UTF8Encoding(false)` 保存 XML。
- `Program.cs` 生成默认 `App.config` 改用 `XmlConfigFile.Save(...)`。
- `AppConfigWriter.cs` 保存 GUI 配置改用 `XmlConfigFile.Save(...)`。
- 发布脚本现在会检查发布目录 `App.config` 不能带 UTF-8 BOM。

最终验证：
- 脚本 smoke test 连续运行 9 个会写配置的脚本，全部退出码 0。
- 每次运行后 `App.config` 首字节仍为 `3C 3F 78 6D 6C 20 76 65`。
- 运行后按 UTF-8 读取仍能找到中文注释 `RFID/ERack串口硬件配置`。

### 2.4 Zebra status 参数错误

历史归档事实：
- `notes\20260624-二期远程桌面联调完整归档.md` 已写明：Zebra SDK `status` 命令一次只能查一种，`--printer` 或 `--portstatus` 不能同时传。

本轮错误：
- 初版代码把 `status ... --printer --portstatus --verbose` 放在同一次调用里。
- 这和历史归档冲突。

修正：
- `ZebraCommandLinePrinterGateway` 已改成两次调用：
  - `status ... --printer --verbose`
  - `status ... --portstatus --verbose`
- 两次原始输出都只写日志。
- S8F4/S6F11 对外描述仍只返回中文：
  - `打印成功`
  - `打印机脱机`
  - `缺纸`
  - `内容为空`
- `zebra-status-driver.cmd` 也改成两次 status 调用，避免诊断脚本误导。

### 2.5 客户脚本在非交互环境下不稳定

问题 1：
- `timeout /t 3 /nobreak >nul` 在当前非交互 smoke test 中报 `Input redirection is not supported`。

修正：
- 客户脚本尾部等待统一改为：
  - `ping -n 4 127.0.0.1 >nul`
- 效果：仍然短暂停留后自动关闭，不依赖 stdin。

问题 2：
- `set-profile-real-unit-client.cmd` 在 smoke test 中出现 `The system cannot find the batch label specified - update_config`。

修正：
- 该脚本重写为无 batch 子程序版本，不再使用 `call :update_config`。
- 单独验证和全量 smoke test 均通过，退出码 0。
