二期需求：
多 LOCATION_ID -> COM + DeviceAddress 路由，Location ID 与 MCU 一对一。GUI 可 Discover/Save Zebra USB 打印机地址，保存到 App.config 的 Printer:ZebraPrinterAddress。RFID 写入支持 1-32 个 ASCII 字符；非 8 倍数时先读旧 tag，保留尾部旧字节，再按 8 字节页写回。S5F11 可选第三参数 ReadLengthBytes，旧两字段兼容，默认 32。Runtime:Mode=Unit|Server|Both；Server TCP 真实实现等客户参考代码。传感器由 MCU 内部持续检测，但 MCU 不会主动向电脑串口推送状态，电脑程序需要 500ms 定时发送 0x05 轮询。读取 data 段第二个字节，sensorState == CheckLevel 表示有货，sensorState != CheckLevel 表示无货；CheckLevel 默认 0 且必须可配置。无货清屏发送 0x14 且 data 段 0 字节。有货读到 RFID 后显示 RFID 并发 S6F21；有货读不到 RFID 时显示 NO ID  2001，并发 S6F21，RFID 字段为空、状态为上架。S6F11 写 RFID 结果是自定义 report，字段包含 RFID 值、结果码、描述等；S10F12 仍是写 RFID 即时应答。S5F1 报警暂不默认发送，等客户给完整消息格式。UI 低分辨率自适应，目标至少 1024x768 可用。标签布局按 60mm x 40mm、203dpi 起点调参，WidthDots=480，HeightDots=320。

ERACK Server/Unit 当前状态：
客户未提供 Server 与多个 Unit 之间的正式协议。2026-06-23 已先自定 TCP JSON Lines 协议并实现。Server 默认监听 127.0.0.1:7801，Unit 主动连接并 RegisterUnit，Server 按 ShelfId 路由 S5F11/S10F11/S8F3 到 Unit。Unit 执行本机硬件/打印后返回结果，并把 ShelfStateChanged/RfidWriteEvent/PrintEvent 主动发给 Server，由 Server 转成 S6F21/S6F11 发 MES。Runtime:Mode=Server 启动 SECS+Server，不碰本机硬件；Runtime:Mode=Both 同进程启动 Server 和本机 Unit Client 用于单机联调；Runtime:Mode=Unit 默认仍是单机直连 MES，若 ERackClient:Enabled=true 则作为 Unit Client 连外部 Server 且不启动本机 SECS。

2026-06-24 已新增 ERackSimulation，用于远程桌面无 MCU/无 Zebra 时先测单机 Server+Client 链路。默认 ERackSimulation:Enabled=false。单机联调时改 Runtime:Mode=Both、ERackServer:Enabled=true、ERackClient:Enabled=true、ERackHardware:Enabled=false、Printer:RealPrintEnabled=false、ERackSensorDisplay:Enabled=false、ERackSimulation:Enabled=true。仿真 Worker 默认每 5 秒循环发布三种 ShelfStateChanged：有货且有 RFID、有货但无 RFID、无货；Both/Client 模式下事件会先从 Unit Client 发到 Server，再由 Server 转成 S6F21。MockHardwareGateway 已改成记住 S10F11 写入的 tag，后续 S5F11 可读回，用于验证 Server 路由。

关键沟通口径：
code=7 是硬件/串口执行异常通用码，不是 SECS/GEM 协议层错误，不回 S9Fx。写 tag code=7 会在 S10F12 返回 ResultCode=7，Description 带具体异常。读 RFID code=7 会在 S5F12 返回 ResultCode=7；当前 S5F12 无 Description 字段，详情看日志。写 tag 等待时间约 4.2 秒：200ms + 20*200ms。读 RFID 等待时间约 1 秒：50ms + 20*50ms。code=2 是参数校验失败，例如旧版本 tag 长度不是 8/16/24/32，通常不会发硬件命令。如果现场还提示 8/16/24/32，说明现场仍是旧写 tag 校验规则或在用客户 ERack 测试程序，不是硬件问题。
