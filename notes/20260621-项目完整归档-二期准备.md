一、项目一句话
这是一个 Windows 10 x64 / .NET 8 WinForms 中转程序。电脑作为 EQ 侧，通过 SECS/GEM HSMS TCP 接 MES/Host，通过串口接 ERack/MCU/RFID，通过 Zebra SDK USB Direct 发送 ZPL 到 Zebra 打印机。正式入口是双击 PrinterSecsGem.Eq.exe，GUI 和 SECS/GEM 后台在同一进程运行，Host 指令和 GUI 按钮共用同一个 ERackSerialHardwareGateway。

不要再讨论 GUI 模式和 run.cmd 二选一。run.cmd 只是备用纯后台模式。
双击后：打开 WinForms 状态界面；启动 SECS/GEM EQ worker；按 App.config 连接 Host；自动打开 RFID COM；接收 Host 的 SECS 指令；通过同一个串口实例操作 RFID；通过 Zebra SDK USB Direct 打印。

九、SECS 错误码沟通口径
code=0：成功。
code=2：参数校验失败，例如当前现场旧版本 tag 长度不是 8/16/24/32。
code=3：内容为空或 ASCII 不合法等输入问题。
code=4/5：打印机发现或发送等打印相关错误。
code=6：location 未配置。
code=7：硬件/串口执行异常通用码，例如 COM 打不开、RFID 响应超时、返回数据格式异常。code=7 不是 SECS/GEM 协议层错误，不回 S9Fx，而是在对应业务回复中返回 ResultCode=7。

S10F11 写 tag 失败时回复 S10F12，字段是 ShelfId、LocationId、ResultCode、Description。写 tag 硬件超时时 ResultCode=7，Description 带具体超时文本。
S5F11 读 RFID 失败时回复 S5F12，字段是 ShelfId、Locations、ResultCode。当前 S5F12 没有 Description 字段，失败时 Locations 为空，ResultCode=7，具体原因看日志。
客户截图里前一笔 code=7 是写 tag 等硬件响应超时；后一笔 code=2 是另一笔写 tag 请求长度校验失败。它们不是因果关系，只是日志时间挨得近。

十一、RFID 写入规则
旧规则：写入长度必须是 8、16、24、32 个 ASCII 字符。
原因：RFID 底层按 8 字节页写，旧程序把业务输入也限制为完整页长度。
二期需求：支持 1-32 个 ASCII 字符。
二期实现思路：整页长度直接写；非 8 倍数长度先读旧 tag，再覆盖前 N 字节，保留尾部旧字节，按向上取整后的页长度写回。比如 9 字节按 16 字节写，第 9 字节落到第二页第一个字节，后 7 字节保留旧内容。
如果非整页写入时先读旧 tag 失败，则不发送写入，返回失败。
短写不清尾部，读回 32 字节后后面可能还有旧数据，这是确认后的行为，不是 bug。

二十一、2026-06-23 二期传感器与主动上报确认
客户确认 MCU 不会在状态变化时主动向电脑串口推送数据。传感器由 MCU 硬件内部持续检测，电脑程序需要定时发送 0x05 轮询当前状态。客户推荐轮询周期 500ms。
传感器读取使用 0x05，取 data 段第二个字节作为 sensorState。客户原代码里 checkLevel 可配置，之前配置为 0。当前确认按 checkLevel 语义实现：sensorState == checkLevel 表示有货，sensorState != checkLevel 表示无货。现场有 3/4 种传感器，有的判高、有的判低，所以 CheckLevel 必须保留为 App.config 配置项。
无货时清空显示屏：发送 0x14，data 段 0 字节。不要把空文本转换成 0D 0A + 16 个 0x7F。
有货时读取 RFID。读到 RFID 时显示 RFID，并主动发 S6F21：ShelfId、LocationId、RFID、状态 1、Unix 秒时间戳。读不到 RFID 时显示 NO ID 后加失败码，当前默认 NO ID  2001；同时也主动发 S6F21，RFID ID 字段为空字符串，状态 1。无货时发 S6F21，RFID ID 为空字符串，状态 0。
电子货架 SECS/GEM HSMS 协议文档 20260306.pdf 已确认：S6F21 是 EQ->Host 格口状态变化主动上报；S5F11/S5F12 是 Host 主动查询和 EQ 应答；S8F3/S8F4 是打印；S10F11/S10F12 是设置 RFID ID。
客户图中 CEID：CE2001=格口状态变更关联 S6F21；CE2002=RFID ID 设置成功关联 S6F11；CE2003=打印完成关联 S6F11；CE2004=打印失败关联 S6F11。ALID：AL2001=格口 RFID 读取失败；AL2002=打印机离线；AL2003=RFID ID 设置失败；AL2004=格口号不存在。
本次理解“6f11 写 RFID，返回成功或失败报告字段（自定义的）：RFID 的值”：S6F11 是通用事件上报，写 RFID 成功/失败都可上报一个自定义 report，report 字段至少包含 ShelfId、LocationId、RFID 值、ResultCode、Description、Timestamp。S10F12 仍是 Host 写 RFID 命令的即时应答，S6F11 是额外主动事件，不替代 S10F12。
S5F1 报警暂不默认发送，因为目前只有 ALID 表，没有完整 S5F1 消息格式。后续如果客户确认格式，再接 AL2001/AL2002/AL2003/AL2004。

二十三、2026-06-23 ERACK Server/Unit 自定协议实现
客户没有提供 ERACK Server 与多个 ERACK Unit 之间的详细协议。为先跑通整体功能，当前先自定 TCP JSON Lines 协议，后续客户给正式协议时只替换 ErackNetwork 适配层。
运行模式：
Runtime:Mode=Unit：本机单元模式，MES 可直连本机 SECS/GEM，本机直接控制 MCU/RFID/显示屏/Zebra。若 ERackClient:Enabled=true，则作为 Unit Client 主动连接 ERACK Server，此时不启动本机 SECS，避免同机端口冲突。
Runtime:Mode=Server：ERACK Server 模式，启动 SECS/GEM 面向 MES，并启动 TCP listener 接收 Unit 注册；不启用本机硬件/传感器。
Runtime:Mode=Both：单进程联调模式，同时启动 Server 和本机 Unit Client，Unit Client 默认连 127.0.0.1:7801 注册 SHELF001；MES 连接该进程的 SECS/GEM，命令仍经 Server 路由到本机 Unit Client。
自定 TCP 默认：Server 监听 127.0.0.1:7801；Unit 主动连接 Server；请求超时 10000ms；Unit 断线后 3000ms 重连；心跳 5000ms。
JSON Lines 外壳字段固定为 MessageId、MessageType、ShelfId、Payload、Timestamp。请求和响应用同一 MessageId 匹配。
Unit 首包必须是 RegisterUnit，payload 包含 UnitId、ShelfId、Locations。Server 保存 ShelfId -> UnitConnection 路由。
Server 可转发三类请求：ReadShelfStatus、WriteRfid、Print。Unit 执行本机 IHardwareGateway/IPrinterGateway 后返回对应 response。

二十四、2026-06-24 单机 Server+Client 仿真补充
为远程桌面环境先验证 Server/Unit 链路，新增 ERackSimulationOptions 和 ERackSimulationWorker。配置节为 ERackSimulation，默认 Enabled=false，不影响现场真实硬件运行。
ERackSimulation 打开后只在 Runtime:Mode 启用 Unit 时运行；它不访问真实 MCU、RFID 或打印机，只按配置的 ShelfId、LocationId、Tag 周期性发布 ShelfStateChanged 事件。默认流程：启动延迟 3000ms 后发布“有货且有 RFID”，间隔 5000ms 后发布“有货但无 RFID”，再间隔 5000ms 发布“无货”，Loop=true 时循环。事件通过 IERackEventSink 发送，所以 Both/Client 模式会先经 Unit Client 发到 Server，再由 Server 转成 S6F21；普通 Unit 模式则直接转成 S6F21。
新增配置默认值：ERackSimulation:Enabled=false，ShelfId=SHELF001，LocationId=LOC001，Tag=RFID1234567890，StartupDelayMilliseconds=3000，StepIntervalMilliseconds=5000，Loop=true。
MockHardwareGateway 已增强：S10F11 通过 Mock 写入后，会在内存里记住同一 ShelfId+LocationId 的 tag；后续 S5F11 查询会返回刚写入的值，并按 ReadLengthBytes 截断。这样无 MCU 测试时可以验证 S10F11 -> Server -> Unit -> MockHardware -> S10F12，再用 S5F11 读回确认路由。
远程桌面单机联调推荐配置：Runtime:Mode=Both，ERackServer:Enabled=true，ERackClient:Enabled=true，ERackClient:ShelfId=SHELF001，ERackHardware:Enabled=false，Printer:RealPrintEnabled=false，ERackSensorDisplay:Enabled=false，ERackSimulation:Enabled=true。SECS Simulator/MES 只连这个进程的 SECS 端口；ERACK Unit Client 也在同进程内连接 127.0.0.1:7801。
