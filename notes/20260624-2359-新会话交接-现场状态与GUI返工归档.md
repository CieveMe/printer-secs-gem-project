## 4. App.config 编码与现场配置规则

关键事实：

- 最小包没有 `App.config`，所以单纯覆盖最小包不会改变现场 `App.config` 的编码。
- 如果现场 `App.config` 已经乱码或编码需要修复，必须在覆盖最小包后运行：

```bat
run-after-copy-repair-app-config.cmd
```

该脚本设计目的：

1. 备份现场原 `App.config`。
2. 用 `App.config.reference` 重建 UTF-8 无 BOM 且中文注释正常的配置文件。
3. 将现场旧配置中同名 key 的值迁移回新文件。

注意：

- 这是为了避免直接覆盖现场 COM/SECS/Zebra 配置。
- 如果用户问“都覆盖了还怎么备份现场 App.config”，正确解释是：最小包本身不覆盖 `App.config`，所以脚本运行时仍能读到旧的现场 `App.config` 并先备份。
- 完整包只适合新建目录完整部署，不适合覆盖旧现场目录。
- 如果已经用完整包覆盖了旧目录，那旧现场 `App.config` 已经被覆盖，除非之前另有备份，否则软件侧无法再恢复旧值。

## 7. SECS 中文 Description 编码问题

已知现象：

- S6F11 的 Description 曾显示为 `????`。
- 这很可能是因为当前 SECS 字段用的是 ASCII `A(...)`，中文无法在模拟器/Host 正常显示。

已知技术事实：

- secs4net 里存在：
  - `A(string)`
  - `J(string)`
  - `SecsFormat.JIS8`

下一轮需要决策并验证：

1. 如果客户 Host 和模拟器支持 JIS8，就把协议里的中文 Description 从 `A(description)` 改为 `J(description)`，至少覆盖：
   - S8F4 Description
   - S10F12 Description
   - S6F11 RFID write Description
   - S6F11 print Description
2. 如果客户 Host 只接受 ASCII，就不要在 SECS Description 里放中文。改成 ASCII 枚举，例如：
   - `PRINT_SUCCESS`
   - `PRINTER_OFFLINE`
   - `PAPER_EMPTY`
   - `CONTENT_EMPTY`
   GUI 和日志继续显示中文。

不要再无验证地承诺 `A="打印成功"` 一定能显示中文。当前证据显示它会变成问号。
