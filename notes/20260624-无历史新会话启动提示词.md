当前已实现的关键二期能力：
- Runtime:Mode=Unit|Server|Both。
- Runtime:Mode=Both 时，一台电脑只启动一次 PrinterSecsGem.Eq.exe，同进程内同时跑 Server 和本机 Unit Client。
- Runtime:Mode=Server 时支持一台电脑只有服务端没有客户端。
- 自定 TCP JSON Lines ERACK Server/Unit 协议：Unit 主动连 Server，默认 127.0.0.1:7801，RegisterUnit 注册 ShelfId，Server 按 ShelfId 路由 S5F11/S10F11/S8F3。
- Unit 主动事件 ShelfStateChanged/RfidWriteEvent/PrintEvent 会发给 Server，Server 转成 S6F21/S6F11。
- 传感器真实逻辑：MCU 不主动推送，电脑 500ms 轮询 0x05，取 data 第二字节，sensorState == CheckLevel 表示有货，CheckLevel 默认 0 可配置；无货清屏发 0x14 且 data 段 0 字节。
- RFID 写入二期规则：支持 1-32 ASCII；非 8 倍数先读旧 tag，再覆盖前 N 字节并保留尾部旧内容。
- S5F11 支持可选第三参数 ReadLengthBytes，旧两字段兼容，默认 32。
- GUI 可 Discover/Save Zebra USB 打印机地址到 App.config。
- 新增 ERackSimulation，用于无 MCU/无 Zebra 远程仿真测试。
- MockHardwareGateway 已增强：S10F11 写入后，后续 S5F11 可读回刚写入 tag。

5. 验证 S1F1/S1F3。
6. 发 S5F11 SHELF001 LOC001，期望 S5F12 code=0。
7. 发 S10F11 SHELF001 LOC001 TEST123456，期望 S10F12 code=0，并看到 S6F11。
8. 再发 S5F11，应读回 TEST123456。
9. 发 S8F3 SHELF001 PRINTER001 TEST123456 1，期望 S8F4 code=0，并在 output/zpl 生成 ZPL，随后看到 S6F11。
10. 等待仿真 S6F21 循环上报：有货有 RFID、有货无 RFID、无货。

错误码口径：
code=0 成功。
code=2 参数校验失败，例如旧版本 tag 长度不是 8/16/24/32。
code=6 ShelfId/Unit/Location 路由找不到或 Unit 不在线。
code=7 硬件/串口执行异常通用码，不是 SECS/GEM 协议层错误，不回 S9Fx，而是在 S10F12/S5F12 等业务回复里返回 ResultCode=7。
