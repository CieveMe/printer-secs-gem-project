# 电子货架 SECS/GEM-HSMS 协议摘要

来源文件：`电子货架SECSGEM-HSMS协议文档20260306.pdf`

## 基本角色

按当前文档理解：

- Host：外部上位系统，负责查询、下发打印、设置 RFID。
- EQ：电子货架/本项目程序侧，负责应答、上报格口状态、事件和报警。

该角色划分仍需进一步确认，尤其是最终软件部署在哪台设备上、由谁主动建立 HSMS 连接。

## 基础消息

| 消息 | 方向 | 说明 | 应答 |
| --- | --- | --- | --- |
| S1F1 W | Host -> EQ | 心跳询问 | S1F2 |
| S1F3 W | Host -> EQ | 读取设备整体状态 | S1F4 |
| S5F1 | Host -> EQ | 报警事件上报，文档方向需复核 | - |
| S6F11 | EQ -> Host | 通用事件上报 | - |

## 业务消息

### 查询格口状态

- `S5F11 W`：Host 查询指定货架/格口状态。
- `S5F12`：EQ 返回货架编号、格口列表、RFID、上下架状态、结果码。
- `S6F21`：EQ 主动上报格口状态变化。

### 下发打印

- `S8F3 W`：Host 下发打印指令。
- `S8F4`：EQ 返回打印结果。

字段包括：

- 电子货架编号，如 `SHELF001`
- 打印机编号，如 `PRINTER001`
- 打印内容，如 `SHELF001-LOC001-RFID1234567890`
- 打印份数

### 设置 RFID

- `S10F11 W`：Host 设置指定格口 RFID。
- `S10F12`：EQ 返回设置结果。

已确认该流程对应“写 tag”：Host 下发 RFID/tag，EQ 通过串口硬件写入后返回成功或失败。

### 回传 tag

已确认“回传 tag”为 EQ 通过串口硬件读取到 RFID/tag 后主动上报。

当前可按两种消息保留：

- `S6F11`：通用事件上报。
- `S6F21`：格口状态变化主动上报。

最终使用哪条，以联调 Host 侧支持为准。

## 事件与报警

CEID：

- `CE2001`：格口状态变更，上架/下架。
- `CE2002`：RFID ID 设置成功。
- `CE2003`：打印完成。
- `CE2004`：打印失败，缺纸/离线。

ALID：

- `AL2001`：格口 RFID 读取失败。
- `AL2002`：打印机离线。
- `AL2003`：RFID ID 设置失败。
- `AL2004`：格口号不存在。

## 待复核点

- `S5F1` 通常在 SECS 中偏报警上报语义，文档里写成 Host -> EQ，需确认是否为项目自定义。
- `S6F21` 不是最常见的标准事件消息形式，需确认对接 Host 是否按此实现。
- 需要确认 HSMS 主动/被动端、IP、端口、Device ID、T3/T5/T6/T7/T8 超时参数。
- 需要真实报文样例或联调模拟器。
