# 本地验证指南

## 1. 开发环境

建议使用：

- Windows 10/11 x64
- Visual Studio 2022
- .NET 8 SDK

现场系统已确认只考虑 Win10 以上；Win7 兼容不纳入第一阶段。

当前仓库可直接打开：

```text
PrinterSecsGem.sln
```

如果使用命令行：

```powershell
dotnet restore .\PrinterSecsGem.sln
dotnet build .\PrinterSecsGem.sln
```

也可以使用仓库脚本：

```powershell
cd .\scripts
.\build.ps1
```

生成 Win10 x64 测试包：

```powershell
cd .\scripts
.\publish-win-x64.ps1
```

输出目录：

```text
publish/win-x64/
```

现场配置优先修改发布目录里的 `App.config`。程序仍保留 `appsettings.json` 作为默认配置，但同名配置以 `App.config` 为准。

常用连接配置：

```xml
<add key="secs4net:IpAddress" value="127.0.0.1" />
<add key="secs4net:Port" value="5000" />
<add key="secs4net:DeviceId" value="0" />
```

## 2. 不连接 Host 的本地验证

先跑本地 mock 闭环，确认程序、配置、ZPL 模板、硬件接口抽象都能工作：

```powershell
dotnet run --project .\src\PrinterSecsGem.Eq\PrinterSecsGem.Eq.csproj -- --validate-local
```

也可以使用脚本：

```powershell
cd .\scripts
.\validate-local.ps1
```

预期结果：

- 日志显示 local validation completed successfully。
- 生成 ZPL 文件到 `src/PrinterSecsGem.Eq/output/zpl/`。
- 写入运行日志到 `src/PrinterSecsGem.Eq/logs/printer-secs-gem.log`。
- mock 写 tag 成功。
- mock 查询格口状态成功。
- 能构造标准 `S6F11` 主动上报消息。

如果只需要改本地验证内容，修改 `appsettings.json`：

```json
{
  "LocalValidation": {
    "ShelfId": "SHELF001",
    "LocationId": "LOC001",
    "PrinterId": "PRINTER001",
    "Content": "EFS08IZS",
    "Copies": 1
  }
}
```

## 3. 配置 EQ 主动连接 Host

修改：

```text
src/PrinterSecsGem.Eq/appsettings.json
```

重点字段：

```json
{
  "secs4net": {
    "DeviceId": 0,
    "IsActive": true,
    "IpAddress": "127.0.0.1",
    "Port": 5000
  }
}
```

`IsActive: true` 表示本程序作为 EQ 主动连接 Host。Host 侧需要以 Passive/Server 方式监听对应端口。

Host IP、端口、Device ID 已做成配置项。本地开发时可以用默认值或模拟值；真实联调/部署时再按现场 Host 填写。

## 4. 验证 SECS 主流程

推荐先使用 SECS/HSMS 模拟器，或者使用 `secs4net` 官方 sample 做 Host 侧测试。

本程序当前已处理：

- `S1F1 -> S1F2`
- `S1F3 -> S1F4`
- `S5F11 -> S5F12`
- `S8F3 -> S8F4`
- `S10F11 -> S10F12`

第一阶段重点验证：

```text
Host 下发打印内容 -> EQ 收到后生成 ZPL -> EQ 回复成功/失败
```

对应消息：

```text
S8F3 -> S8F4
```

样例报文：

```text
samples/secs/S8F3-print.sml
samples/secs/S10F11-write-tag.sml
samples/secs/S5F11-query-shelf.sml
```

## 5. 验证打印模板

当前打印实现是模拟方式，不直接连打印机，而是把 ZPL 写到：

```text
output/zpl/
```

可先用 `samples/zpl/EFS08IZS.zpl` 检查版式，目标效果是：

- 上方大号文本
- 中间 Code128 一维码
- 下方可读文本

当前已确认：

- 标签尺寸：2.5 x 1.575 英寸，约 63.5mm x 40mm。
- 打印机 DPI：203dpi。
- ZPL 画布：约 508 x 320 dots。

后续接入 Zebra USB 打印时，把 `FilePrinterGateway` 替换为实际实现即可。

## 6. Tag 读写确认

已确认的业务映射：

- 写 tag：Host 下发 RFID/tag，EQ 通过串口硬件写入，返回结果，对应 `S10F11/S10F12`。
- 回传 tag：EQ 读取到 RFID/tag 后主动上报，对应标准 `S6F11`。

当前代码中：

- `S10F11/S10F12` 已接入 `IHardwareGateway.WriteTagAsync`。
- `S6F11` 的消息构造在 `SecsEventMessageFactory`，等实际串口读 tag 触发点确定后接入发送逻辑。
- 串口硬件代码由对方提供 C# 源码、DLL、示例工程或现有项目片段，本项目先分析再集成；代码能看出来的调用方法、入参、返回值、错误码不提前追问。

## 7. 远程联调反馈要求

真实打印机、串口硬件、Host 环境在现场时，每次拉代码测试后至少反馈：

- 当前使用的配置文件，注意可隐藏敏感 IP。
- 程序完整日志。
- 运行命令或启动方式。
- 打印结果照片。
- 报错截图。
- 如果是 Host 对接问题，提供对应时间点的 Host 侧现象或报文日志。

只反馈“没反应”“打印不了”无法定位问题。

程序日志统一走 `log4net`，默认同时输出到控制台和文件：

```text
logs/printer-secs-gem.log
```

日志路径、滚动策略和格式在 `src/PrinterSecsGem.Eq/log4net.config` 里修改。
`Secs4Net.Sml` 暂不强制引入；当前先保证 SECS 报文相关日志能清楚打印，后续如果需要 SML 格式再补。

如果使用 `publish/win-x64` 目录测试，可以直接运行：

```powershell
.\PrinterSecsGem.Eq.exe --validate-local
```

真实联调时去掉 `--validate-local`，程序会按 `appsettings.json` 连接 Host。

## 8. 交付前检查

至少完成以下检查再交付：

- 能成功连接 Host。
- Host 下发 `S8F3` 后能生成 ZPL 并回复 `S8F4`。
- Host 下发 `S10F11` 后能调用串口写 tag 接口并回复 `S10F12`。
- 串口读到 tag 后能触发标准 `S6F11` 主动上报。
- 打印机实机能输出目标标签。
- 日志能定位连接失败、打印失败、串口失败。
